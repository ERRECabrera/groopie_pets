La aplicación requiere dos modelos: User y Pet, con una relación de 1:n entre ambas.
En un principio se había diseñado para ser una single page sin necesidad de recarga con controladores que prácticamente se comportaban como una API, pero esto generó problemas con la gema paperclip. Era imposible actualizar la foto de la mascota sin hacer un submit.
Aunque el foco no estaba en el front se ha intentado también desarrollar mínimamente esta parte para practicar y ampliar conocimientos adquiridos tanto en el diseño de la interfaz de usuario como en la gestión de eventos en javascript.
Finálmente, se han realizado las modificaciones que el personal de Groopify tuvo a bien sugerirme.
